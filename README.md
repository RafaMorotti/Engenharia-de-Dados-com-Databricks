# Engenharia de Dados com Databricks

## 1. Objetivo do projeto

Este projeto tem como objetivo praticar comandos em Python com a API PySpark, além de desenvolver familiaridade com a ferramenta Databricks.

## 2. Estrutura do repositório

O repositório contém dois notebooks iniciais:

- um notebook para a criação de pastas, diretórios e catálogo;
- um notebook para a criação das tabelas da camada bronze.

Além disso, o repositório inclui os arquivos-fonte utilizados para a construção da camada bronze.

A camada bronze foi criada em formato de tabelas, e o código responsável por sua criação está no notebook **"Criação de tabelas na camada bronze"**.

## 3. Ordem de execução dos notebooks

Para reproduzir o projeto, execute os notebooks na seguinte ordem:

1. **Criação estrutura**
2. **Criação de tabelas camada bronze**

## 4. Explicação da camada bronze/raw

A camada bronze contém os dados em formato de tabela, sem tratamento. Nessa etapa, os dados permanecem brutos, servindo como base para a construção da camada silver.
